### AC2 — No spawn toast pile-up: FAIL (transient race during the load reveal)
- Steady state is correct: fresh profile at spawn, `data-focused` set (plato slug), `#museum-hint` visible at (320,806 640x62), `#museum-prompt` hidden (`display:none`, rect 0x0) — r10-spawn-fresh-1280.png. After a keypress dismisses the hint, the prompt appears at (401,826 478x42) for the still-focused painting — r10-spawn-afterdismiss-1280.png.
- BUT during the dark-to-light reveal the prompt renders visible AND overlapping the hint for 1-2 frames: captured organically in a screenshot-only run mid-fade (r10-fade-mid-0-lum123.png — amber prompt pill sitting on top of the garbled hint text, the exact image of the original blocker), and reproduced 2/2 in DOM polling runs (r10j: both-visible + rect-overlap samples at t=4184ms; t=4109-4237ms; ~130-250ms at this environment's frame rate, loaded flips at ~3.3s). The AC requires "the prompt stays hidden while the hint shows"; it does not.
- Mechanism (for the fix, confirmed in source after measurement): `focusStore.hintVisible` initializes `false` (`src/components/museum/focusStore.ts:10`) and only flips true in the Hint's post-mount effect (`src/components/museum/MuseumRoom.tsx:60-72`). The per-frame prompt visibility check `best && !focusStore.hintVisible` (MuseumRoom.tsx:306-307) runs on the first textured frames before that effect lands, so on every fresh visit the prompt paints over the hint until the next frame. Load-time frames are the jankiest on any hardware, so this is plausibly visible outside the eval environment too.
- Fix: make the fresh-profile hint state known before the first frame. Either initialize the store lazily, e.g. in `focusStore.ts`: `hintVisible: typeof window !== "undefined" && (() => { try { return !localStorage.getItem("museum.hintSeen.v1"); } catch { return true; } })()`, or set `focusStore.hintVisible` inside the Hint's `useState` initializer / a `useLayoutEffect` instead of `useEffect`. Verify: fresh profile, screenshot every frame across the reveal; the prompt must never be visible in any frame where the hint is visible.
